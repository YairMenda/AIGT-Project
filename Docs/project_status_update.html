Project Status Update

Project Title: Comparative Analysis of Counterfactual Regret Minimization (CFR) Variants in Kuhn Poker

Students: Yair Menda (207273558), Roy Houber (322274259)

Date: January 08, 2026

(a) What has been done so far

We have successfully completed the implementation phase of our proposed research and have begun the analysis phase. Key milestones achieved include:

  • Done Game Engine Implementation: Developed a fully functional custom simulator for Kuhn Poker, including game tree generation, information set handling, and terminal state evaluation.
  • Done Algorithm Implementation: Implemented all proposed solver variants to ensure comprehensive comparison:
    • Vanilla CFR (Baseline)
    • CFR+ (with regret flooring and alternating updates)
    • Discounted CFR (DCFR) with configurable weighting parameters
    • Linear CFR
    • Softmax CFR (Boltzmann exploration)
  • Done Extended Scope: Implemented additional experimental solvers beyond the original proposal, including Quadratic CFR, Exponential CFR, and Pruned CFR.
  • Done Analysis Framework: Built a robust benchmarking suite (analysis.py) that tracks:
    • Exploitability (Nash distance) against the known Game Value (-1/18).
    • Convergence speed relative to both Iterations and CPU/Wall-clock time.
    • Strategy profile stability and variance at key decision points.
    • Memory usage estimates for each solver.
  • Done Preliminary Results: Generated initial comparative graphs confirming that CFR+ and DCFR significantly outperform the Vanilla CFR baseline in convergence speed.

(b) What is left to do

Our focus now shifts to deep analysis and final reporting. The remaining tasks are:

  • To Do High-Fidelity Experiments: Run long-duration simulations to observe tail-end convergence behavior and verify stability for all variants over large iteration counts ($T$).
  • To Do Strategy Profile Analysis: Verify if solvers converge to theoretically identical GTO frequencies (e.g., verifying Player 1 checks a King to trap or bets a Jack ~1/3 of the time).
  • To Do Parameter Sensitivity: Conduct a grid search to test the sensitivity of DCFR to its discount parameters ($\alpha, \beta, \gamma$) and Linear weighting schemes.
  • To Do Final Report: Synthesize all findings into the final project report, interpreting the theoretical implications of the empirical results and computational efficiency.
